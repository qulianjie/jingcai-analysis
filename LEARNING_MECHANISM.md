# 竞彩预测算法 — 学习方法与预测机制

> 本文档详细说明反馈学习引擎如何从历史数据学习模式，
> 以及最终结论引擎如何利用这些模式进行预测和信心度修正。

---

## 一、反馈学习引擎 (`feedback_learner.py`)

### 1.1 数据来源

| 数据源 | 路径 | 说明 |
|--------|------|------|
| `feedback.json` | `learnings/feedback.json` | 反馈闭环写入的赛后数据（实际比分、预测是否正确） |
| `scores_history.json` | `learnings/scores_history.json` | 比分历史记录 |
| 各场比赛的最终报告 | `tasks/{日期}/周*.md` | 从报告中提取组合特征（欧赔盘路、亚盘、让球等） |
| `step25_zhuangjia.json` | match 目录下 | 庄家盈亏方向（大热方/主胜盈亏/客胜盈亏） |
| `step26_profit_ratio.json` | match 目录下 | 盈亏比例数据（投注占比、各方向盈利率） |

### 1.2 特征提取流程

对每场有反馈的比赛，`extract_combo_from_report()` 从最终报告提取以下特征：

**单维度特征（从报告中解析）：**
1. **澳门亚盘** — 具体盘口（如"半球""平手/半球"）
2. **欧赔盘路** — 竞彩/Interwetten/百家各家的升/降趋势（⬇⬆⬆ 等）
3. **让球盘路** — Step5 让球同赔的盘路变化方向
4. **澳门亚盘同赔分布** — Step7 澳门同赔的盘口变化类型统计
5. **各维度信号** — 报告"各维度信号明细"表格的 10 个维度的分值+方向+权重
6. **盘路匹配度** — 高/中/低
7. **欧赔数值** — 竞彩官方初盘/即时盘的胜平负数值
8. **综合信心** — 报告给出的信心百分比

**补充数据（Step25/26）：**
- 庄家盈亏方向（主胜/平局/客胜哪个方向盈利）
- 投注占比（主胜/平局/客胜的投注百分比）
- 盈亏占比（各方向盈利/亏损比例）

### 1.3 组合标签生成

从上述特征按以下规则**自动组合成标签**（每场比赛生成上百个标签）：

**一维标签：**
```
欧赔:利好客
让球:利好主  
亚盘:中立
百家:利好客
盘路:高
庄家:主胜
联赛:日职
信心:中(55-69%)
澳门亚盘:半球
竞彩盘路:升降降
IW盘路:升升降
百家盘路:升降降
投注占比精确:胜30%平20%负50%
庄家盈亏:胜赢平亏负赢
```

**二维标签（逐个维度配对 × 联赛）：**
```
欧赔:利好客×联赛:日职
让球:利好主×联赛:日职  
盘路:高×联赛:日职
澳门亚盘:半球×竞彩盘路:升降降
竞彩盘路:升降降×投注占比精确:胜30%平20%负50%
```

**三维标签：**
```
欧赔:利好客×让球:利好主×联赛:日职
欧赔:利好客×亚盘:中立×联赛:日职
庄家:主胜×盘路:高×联赛:日职
澳门亚盘:半球×竞彩盘路:升降降×庄家盈亏:胜赢平亏负赢
```

**四维标签：**
```
欧赔:利好客×让球:利好主×亚盘:中立×联赛:日职
欧赔:利好客×让球:利好主×庄家:主胜×联赛:日职
欧赔:利好客×让球:利好主×亚盘:中立×庄家:主胜×联赛:日职
```

### 1.4 准确率统计

对每个组合标签，统计：
- `total`：匹配该标签的历史比赛场次
- `correct`：其中预测正确的场次

**结果分类：**
- **高准确率组合**：≥5 场且准确率 ≥60%
- **低准确率组合**：≥10 场且准确率 ≤35%

**所有结果保存到** `learned_patterns_v2.json`，包含：
- `high_accuracy_combos`：高准确率组合列表（按准确率排序）
- `low_accuracy_combos`：低准确率组合列表
- `league_accuracy`：各联赛准确率（≥5场）
- `confidence_accuracy`：各信心等级的实际准确率 vs 标称准确率
- `profit_direction_accuracy`：庄家盈亏方向的准确率
- `panlu_accuracy`：盘路匹配度（高/中/低）的准确率

## 二、预测引擎 (`final_conclusion_generator.py`)

### 2.1 基础分值计算（5阶段加权）

```
最终预测分值 = Σ(维度分值 × 维度权重) / 总权重
```

**10 个维度的基础权重分布：**
| 维度 | 默认权重 | 数据来源 |
|------|---------|---------|
| 欧赔趋势 | 15% | step1 欧赔基础 |
| 竞彩同赔 | 12% | step2 竞彩官网同赔 |
| IW同赔 | 10% | step3 Interwetten同赔 |
| 澳门亚盘 | 12% | step6 澳门亚盘 |
| 让球同赔 | 10% | step5 让球同赔 |
| 主队主场 | 10% | step9-13 主队分析 |
| 客队客场 | 10% | step14-18 客队分析 |
| 百家对比 | 10% | step19-23 百家对比 |
| 庄家盈亏 | 5% | step25 庄家盈亏 |
| 盘路匹配 | 6% | step24 盘路匹配 |

每个维度的分值范围 -1.0~1.0，正数利好主队，负数利好客队。

### 2.2 学习模型修正（3层乘法调整）

在基础权重上，学习引擎的产出按以下 3 层叠加修正：

**第1层 - 组合模式匹配（Wilson Score）：**
```
从 learned_patterns_v2.json 的高/低准确率组合中，
匹配当前比赛的上下文标签（联赛/欧赔方向/亚盘方向/让球方向/百家方向/庄家方向/投注占比），
找到最长前缀匹配的 combo → 给出修正因子。
修正因子 = (Wilson(correct,total) + Bayesian(correct,total)) / 2 → trust_to_factor()
```

**第2层 - 联赛准确率（Wilson Score）：**
```
用当前联赛的历史准确率 → Wilson + Bayesian → trust → factor
应用于所有有方向信号的维度权重。
```

**第3层 - 专家盘路模式（expert_patterns.md）：**
```
从 expert_pattern_engine.match() 匹配专家定义的模式规则，
如"法甲 半球→半球平 → 平局倾向"，
命中的模式给出 factor，应用于所有有方向信号的维度权重。
```

最终权重调整公式：
```
调整后权重 = 默认权重 × 模式因子 × 联赛因子 × 专家因子
```

### 2.3 信心度计算

信心度基于以下因素综合：
```
信心度% = |最终分值| × 50 + 维度一致性加分 + 数据质量加分
```

然后经过学习模型再修正：
```
if 学习修正总计 != 0:
    信心度 = max(10, min(95, 信心度 + 联赛修正 + 盘路修正 + 组合标签修正))
```

### 2.4 让球预测

在竞彩方向确定后，从 Step4 让球基础数据和 Step5 让球同赔分析让球方向，生成"让球胜/让球平/让球负"的独立预测。

## 三、学习-预测闭环

```
比赛前                 比赛后                   下次比赛前
   │                     │                        │
   ▼                     ▼                        ▼
┌──────────┐       ┌───────────┐           ┌──────────────┐
│ 流水线运行 │       │ feedback.js │           │  流水线运行    │
│ 24步分析  │──────→│ 获取实际比分  │──────────→│ 读取已学习的   │
│ 生成预测   │       │ 对比预测结果 │           │ learned_     │
└──────────┘       │ 写入feedback│           │ patterns_v2  │
                    │ .json       │           │ → 调整权重    │
                    └──────┬──────┘           └──────────────┘
                           │
                           ▼
                    ┌──────────────┐
                    │feedback_     │
                    │learner.py    │
                    │ 提取比赛特征  │
                    │ 生成组合标签  │
                    │ 统计准确率    │
                    │ 输出 v2.json │
                    └──────────────┘
```

每次 `feedback.js` 跑完（获取赛后数据）→ 手动或自动触发 `feedback_learner.py` → 更新 `learned_patterns_v2.json` → 下次预测时自动使用最新模式。

## 四、专家模式库 (`learnings/expert_patterns.md`)

由连杰提供、基于实战经验总结的盘路变化规律，格式：

```
联赛/玩法 | 盘路变化 | 结果倾向 | 样本量/置信度
```

例：
```
法甲 | 半球→半球平 | 平 | 样本5场
德甲 | 平手受让→半球 | 客胜 | 样本8场
```

`ExpertPatternEngine` 加载后按 match_score 匹配当前比赛，返回 factor 供权重调整。
