## 二、推荐算法：Wilson Score + Bayesian Average

### 为什么选这个组合？

| 方法 | 优点 | 适用场景 |
|------|------|---------|
| **Wilson Score Interval** | 自动惩罚小样本，样本越小置信下界越低 | 判断一个模式是否可靠 |
| **Bayesian Average** | 用全局平均作为先验，样本少时回归均值 | 平滑估计 |
| **组合使用** | Wilson决定"信不信"，Bayesian决定"调多少" | **最佳方案** |

### Wilson Score 公式（95%置信度）

```
Wilson下界 = (p + z²/(2n) - z×√((p(1-p)+z²/(4n))/n)) / (1 + z²/n)

其中：
  p = 准确率 = correct / n
  n = 样本量
  z = 1.96（95%置信度）
```

**实际效果**（z=1.96）：

| 准确率 | 样本量 | Wilson下界 | 说明 |
|--------|--------|-----------|------|
| 80% | 5场 | **33.2%** | 样本太少，实际可能只有33% |
| 80% | 20场 | **59.7%** | 样本适中，下界接近60% |
| 80% | 100场 | **72.2%** | 样本充足，下界接近72% |
| 60% | 200场 | **53.6%** | 大样本稳定，下界53% |
| 0% | 11场 | **0.0%** | 全错就是全错 |
| 100% | 3场 | **43.6%** | 全对但样本太少，不敢信 |

### Bayesian Average 公式

```
Bayesian评分 = (C × m + n × p) / (C + n)

其中：
  C = 伪计数（prior strength），建议取 10（相当于10场的全局平均）
  m = 全局平均准确率（先验），当前约为 0.50（50%）
  n = 实际样本量
  p = 实际准确率
```

**效果**：

| 样本量 | 准确率 | Bayesian评分 | 说明 |
|--------|--------|-------------|------|
| 5场 | 80% | 55.6% | 被全局平均拉回 |
| 20场 | 80% | 66.7% | 逐渐接近实际 |
| 100场 | 80% | 76.9% | 基本信任实际值 |
| 5场 | 0% | 42.1% | 全错也被全局平均拉回一点 |

## 三、完整算法设计

### 核心思路

```
步骤1: 计算每个模式的可信度
  ├─ Wilson下界 → 判断"信不信"
  └─ Bayesian评分 → 平滑后的准确率

步骤2: 映射为权重修正系数
  ├─ Wilson ≥ 0.60 → 增加权重（1.0x ~ 1.5x）
  ├─ 0.40 ≤ Wilson < 0.60 → 不调整（1.0x）
  ├─ 0.20 ≤ Wilson < 0.40 → 轻微惩罚（0.8x）
  ├─ Wilson < 0.20 → 强惩罚（0.5x）
  └─ Wilson = 0.0 → 完全忽略（0.0x）

步骤3: 分层应用（优先级从高到低）
  ├─ 优先级1: 组合模式（三维/四维 combo）→ 最精确
  ├─ 优先级2: 联赛准确率 → 次精确
  ├─ 优先级3: 维度信号准确率 → 较通用
  └─ 优先级4: 信心度修正 → 最通用，最后应用

步骤4: 归一化
  └─ 权重总和 = 1.0，单维范围 5% ~ 30%
```

### 权重修正系数映射

```python
def trust_to_factor(wilson_lower_bound):
    """
    Wilson下界 → 权重修正系数
    
    映射规则：
    - ≥0.60  → 1.0x ~ 1.5x  (高可信，增加权重)
    - 0.40~0.60 → 1.0x       (中等，不调整)
    - 0.20~0.40 → 0.8x~1.0x  (偏低，轻微惩罚)
    - 0.0~0.20  → 0.5x~0.8x  (很低，强惩罚)
    - =0.0      → 0.0x       (零可信，完全忽略)
    """
    if wilson_lower_bound >= 0.60:
        return 1.0 + min(0.5, (wilson_lower_bound - 0.60) / 0.40 * 0.5)
    elif wilson_lower_bound >= 0.40:
        return 1.0  # 中等可信，不调整
    elif wilson_lower_bound >= 0.20:
        return 0.8 + (wilson_lower_bound - 0.20) / 0.20 * 0.2
    elif wilson_lower_bound > 0.0:
        return 0.5 + (wilson_lower_bound / 0.20) * 0.3
    else:
        return 0.0  # 完全忽略
```

## 四、实际效果演示

### 场景1：日职比赛 — 命中高准组合

**当前组合**：`投注占比:高 × 亚盘:利好客 × 联赛:日职`
- 原始准确率：80%（5场）
- **Wilson下界：33.2%** → 属于"偏低"区间
- **修正系数：0.88x**（轻微降低！）

**为什么是降低？** 因为5场80%的Wilson下界只有33%，说明样本太少，不能确信这是规律。算法自动降低了权重，避免了"小样本高准确率陷阱"。

**如果样本量增加到20场，准确率仍为80%**：
- Wilson下界：59.7% → 属于"中等"区间
- 修正系数：1.0x（不调整，因为还不够高准阈值）

**如果样本量增加到100场，准确率80%**：
- Wilson下界：72.2% → 属于"高准"区间
- 修正系数：1.28x（增加权重！）

### 场景2：韩职比赛 — 命中低准组合

**当前组合**：`盈亏占比:低 × 澳门亚盘:利好客`
- 原始准确率：**0%**（11场全错）
- **Wilson下界：0.0%**
- **修正系数：0.0x**（完全忽略！）

**效果**：该维度的权重被归零，自动规避了已知失败的组合。

### 场景3：英超比赛 — 无匹配

- 没有任何匹配的组合
- 所有因子 = 1.0x（不调整）
- 保持原始权重，不引入任何偏差
