# 竞彩25步分析规则

## 一、基础规则

- 数据源优先:主站 https://trade.500.com/jczq/,缺失用 https://live.500.com/ 竞彩足球
- 只采集今日未开赛竞彩足球比赛
- 每次更新数据都必须真实的获取当前实时页面数据,均标注数据更新时间
- 所有URL 禁止出现中文汉字,跟亚盘和澳门相关的一律使用 GBK 编码后 URL
- 严格按顺序执行,不许跳步、不许合并、不许乱关联场次
- 每一步执行完自动继续下一步,不等待、不卡死,需要完成所有24步
- **优先使用Python脚本通过exec执行**:所有数据获取、解析、批量处理优先编写Python脚本通过exec运行,不要使用web_fetch逐页抓取。Python脚本配合requests+BeautifulSoup效率更高、编码更可控。只在Python不可用时才降级使用web_fetch。
- 数据统计完成通过MarkDown表格格式存储
- **最终报告命名规则**:周几00几_比赛球队(如:周四003_莱万特vs塞维利亚)
- **同赔查询统一用初盘参数**:第2步(竞彩欧赔同赔)、第3步(Interwetten同赔)、第5步(让球同赔)、第7步(澳门亚盘同赔)全部使用**初盘**赔率/盘口作为查询参数,不使用即时盘。即:欧赔取初盘胜平负赔率,让球取初盘让球赔率,亚盘取初盘盘口+初盘水位。

## 第0步:获取当日竞彩比赛列表(独立步骤)

**性质**:独立步骤,不与其他分析步骤合并,每场比赛分析前不重复执行

**触发方式**:
- 每天 **上午11:00** 自动执行一次(Cron定时任务)
- 用户手动触发关键词:"跑第0步"、"更新今日竞彩列表"、"获取今天的竞彩数据"

**执行脚本**:`jingcai/step0_fetch_matches.py`

**获取方法**:
```python
import requests, re

headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ...',
    'Accept': 'text/html,application/xhtml+xml,...',
    'Accept-Language': 'zh-CN,zh;q=0.9,en;q=0.8',
    'Accept-Encoding': 'identity',
    'Connection': 'keep-alive',
    'Upgrade-Insecure-Requests': '1',
}

url = f'https://trade.500.com/jczq/?playid=269&g=2&date={日期}'
session = requests.Session()
session.headers.update(headers)
session.get('https://trade.500.com/', timeout=10)  # 先获取Cookie
resp = session.get(url, timeout=15)
html = resp.content.decode('gbk', errors='ignore')
```

**关键细节**:
1. **必须先访问 trade.500.com 首页获取Cookie**,否则返回403
2. **页面同时包含周六和周日比赛**,必须用 `<tr>` 标签的 `data-matchnum` 属性区分(如 `"周日001"`、`"周六026"`)
3. **不能用"周六"/"周日"关键字在HTML中的位置来判断归属**,该方法不可靠
4. **页面编码为GBK**,必须用GBK解码
5. **直接用 requests.get 返回403**,必须用 Session + 完整浏览器请求头

**输出文件**:
- `jingcai/tasks/{日期}/matches_data.json` - 结构化JSON数据
- `jingcai/tasks/{日期}/sunday_matches.md` - Markdown表格

**数据格式**(每场比赛):
```json
{
  "matchnum": "周日001",
  "fid": "1373080",
  "home": "全北现代",
  "away": "浦项制铁",
  "league": "韩职",
  "time": "13:00",
  "rq": "-1"
}
```

**URL参数说明**:
| 参数 | 值 | 说明 |
|------|-----|------|
| playid | 269 | 竞彩胜平负玩法 |
| g | 2 | 分组方式 |
| date | 2026-04-26 | 比赛日期,改此参数可获取任意一天 |

### 八、第8步：相同联赛/杯赛 相同亚盘统计（智能轮次选取）

**⚠️ 第8步与第7步完全无关，是独立步骤**

**执行脚本**:`jingcai/step8_1923_extractor.py`
**用法**:`python step8_1923_extractor.py <home_id> <away_id> <league> <fid> <macau_line> [output_path]`
**示例**:`python step8_1923_extractor.py 2465 848 瑞典超 1362643 "一球/球半" output.txt`

**轮次选取规则**(不包括当前比赛轮次):

| 当前轮次 | 选取策略 | 示例 |
|---------|---------|------|
| **> 20轮** | 从上一轮开始往前推20轮 | 当前35轮 → 取第34~15轮(共20轮) |
| **11~20轮** | 取本赛季已有全部历史轮次 | 当前13轮 → 取第1~12轮(共12轮) |
| **≤ 10轮** | 前一赛季取19轮 + 本赛季第1轮 | 当前2轮 → 取上赛季最后19轮 + 本赛季第1轮(共20轮) |

**数据源**:
- **赛程数据**: `https://liansai.500.com/team/{主队ID}/teamfixture/` (主队历史比赛)
- **亚盘数据**: `https://odds.500.com/fenxi/yazhi-{fid}.shtml` (每场比赛澳门亚盘)
- **欧赔数据**: `https://odds.500.com/fenxi/ouzhi-{fid}.shtml` (每场比赛竞彩欧赔)
- **让球数据**: `https://odds.500.com/fenxi/rangqiu-{fid}.shtml` (每场比赛竞彩让球)

**流程**:
1. **获取整个联赛所有球队赛程**: 从 `liansai.500.com/team/{TEAM_ID}/teamfixture/` 获取16支球队各30场最近赛程
2. **筛选相同联赛**: `SIMPLEGBNAME == 本场比赛联赛名称`
3. **筛选相同盘口**: 精确匹配第6步澳门即时盘口中文名称（如`一球/球半`），去掉"升""降"后缀
4. **去重**: 主队和客队赛程可能包含同一场比赛，需去重
5. **逐场获取澳门亚盘**: 对筛选出的比赛访问 `odds.500.com/fenxi/yazhi-{fid}.shtml` 提取澳门即时盘
6. **采集欧赔和让球数据**: 对选出的比赛采集竞彩官网的欧赔 + 竞彩让球
7. **获取百家欧赔数据**: 对全部联赛比赛访问 `odds.500.com/fenxi/ouzhi-{fid}.shtml` 提取百家初盘
8. **百家匹配**: 只对比**主胜+客胜**（平局不参与），整数+小数点第一位一致即匹配

**⚠️ 跨赛季说明**: 当当前轮次 ≤ 10 时,需要向前一个赛季取数据。注意不同赛季的联赛ID可能不同,需分别获取。

## 八、盘路匹配度判定规则(固化版)

### 1. 欧赔盘路匹配度

**比较对象**:
- 本场比赛:即时胜平负 vs 初盘胜平负的升降变化
- 获取比赛:终盘胜平负 vs 初盘胜平负的升降变化

**判断标准**:
| 匹配度 | 条件 |
|-------|------|
| **高** | 两场比赛的胜/平/负升降变化**完全一致** |
| **中** | 胜平负三项中**部分一致**(如胜降平不变负升) |
| **低** | 胜平负三项**完全不同** |

**示例**:
- 本场比赛:⬆️胜 ⬇️平 ⬇️负 → 获取比赛:⬆️胜 ⬇️平 ⬇️负 → **高**
- 本场比赛:⬆️胜 ⬆️平 ⬇️负 → 获取比赛:⬆️胜 ⬇️平 ⬇️负 → **中**(胜一致,平负不同)
- 本场比赛:⬆️胜 ⬇️平 ➡️负 → 获取比赛:⬇️胜 ⬆️平 ⬇️负 → **低**

---

### 2. 亚盘盘路匹配度

**比较对象**:
- 本场比赛:即时盘口 vs 初盘盘口的升降变化
- 获取比赛:终盘盘口 vs 初盘盘口的升降变化

**判断标准**:
| 匹配度 | 条件 |
|-------|------|
| **高** | 升降变化**完全一致** **且** 本场即时盘 = 获取比赛终盘 |
| **中** | 升降变化**完全一致** **或** 本场即时盘 = 获取比赛终盘(满足其一即可) |
| **低** | 升降变化**完全不同** |

**示例**:
- 本场:半球→平手(降),获取:半球→平手,即时盘=终盘 → **高**
- 本场:半球→平手(降),获取:半球→平手,即时盘≠终盘 → **中**
- 本场:半球→平手(降),获取:半球→半球(不变) → **低**
